# STEP 0A1 — MARKET STRUCTURE CLASSIFICATION

## PURPOSE

Before trading any instrument, determine:

```text id="m02"
what type of market am I actually trading?
```

Because:

* forex ≠ stocks
* stocks ≠ crypto
* spot ≠ derivatives
* exchange ≠ OTC
* real market ≠ synthetic CFD feed

---

# A. STRUCTURE TYPES

| Structure            | Characteristics               | Main Risks            |
| -------------------- | ----------------------------- | --------------------- |
| Interbank FX         | decentralized OTC             | fragmented liquidity  |
| Centralized Exchange | single order book             | auction volatility    |
| Dealer Market        | dealers quote prices          | spread manipulation   |
| OTC Market           | bilateral transactions        | transparency issues   |
| Derivatives Market   | leverage + nonlinear exposure | liquidation cascades  |
| CFD Market           | broker synthetic pricing      | broker execution risk |
| Crypto Exchange      | fragmented global liquidity   | unstable liquidity    |
| Futures Market       | centralized leverage          | forced liquidation    |

---

# B. PRACTICAL USAGE

Before trading ask:

| Question                                   | Example               |
| ------------------------------------------ | --------------------- |
| Is this centralized or fragmented?         | Forex fragmented      |
| Is there real order book transparency?     | Futures yes           |
| Is leverage dominant?                      | Crypto perpetuals yes |
| Is this broker-created synthetic exposure? | CFDs yes              |
| Is liquidity deep or thin?                 | EURUSD deep           |
| Is market open globally or session-based?  | Crypto 24/7           |

---

# C. EXECUTION IMPACT

Different structures require different behavior.

Example:

| Market  | Better Style             |
| ------- | ------------------------ |
| FX      | macro + liquidity        |
| Futures | order flow               |
| Crypto  | volatility + liquidation |
| Stocks  | earnings + flows         |
| CFDs    | execution caution        |

# STEP 0A4 — LIQUIDITY CONSUMPTION ENGINE

## PURPOSE

Price moves because liquidity gets consumed.

NOT because:

```text id="m08"
buyers magically overpower sellers
```

---

# A. CORE PRINCIPLE

```text id="m09"
Price travels toward areas of least liquidity resistance.
```

---

# B. DETECT

| Signal                  | Meaning              |
| ----------------------- | -------------------- |
| spread widening         | liquidity thinning   |
| fast candle expansion   | liquidity vacuum     |
| repeated wick rejection | liquidity absorption |
| thin weekend move       | unstable structure   |
| aggressive reversal     | liquidity exhaustion |

---

# C. PRACTICAL USAGE

Avoid:

* chasing thin breakouts
* entering after exhaustion
* low-liquidity sessions

Prefer:

* active institutional participation
* deep liquidity periods
